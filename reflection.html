Kaalab has performed well as a team, with open communication and constructive collaboration over the course of the project. The chat function of MS Teams has been beneficial for file sharing and clarification of details, with our twice weekly meetings being useful for talking about progress and ideas for the project.

If I had to specify one thing that could have been improved it would be the organization of a clear file structure within teams for the sharing of documents. It can be hard to find things at times, and this is largely due to the team’s lack of familiarity with the program.

I was surprised with the efficiency and usefulness of GitHub as a collaboration tool. I had assumed that with a fairly limited coding aspect to the project it would be used mainly as a hosting tool, but it has been extremely useful to see the project come together in real time through the use of this tool.

I have learned to engage fully with the team and rely on the different skills offered by each member. Minor issues can be ironed out by other members and a different perspective has been useful for refining of content in many instances.
