Clouds, Services, Servers

The IT technology that has been selected is ‘Clouds, services, servers’, which can referred to under the umbrella term ‘cloud computing’. ‘The cloud’ is a remnant of IT industry slang from when technical diagrams of servers and network infrastructure for the Internet were drawn as a cloud. Moving ‘to the cloud’ became a way to refer to computing processes that were taking place on the servers-and-infrastructure part of the Internet. (Cloudflare, 2022)

What does it do?

What is the state of the art of this new technology?

The state of the art of cloud computing as stated by (Zhang, 2010) is described in terms of the key technologies and the current popular cloud computing products.

  • Architectural design of data centres: the design for a cloud computing data centre should endeavour to meet the following goals
    • Uniform high capacity – The “maximum rate of a server-to-server traffic flow should be limited only by the available capacity on the network-interface cards of the sending and receiving servers” (Zhang, 2010)
    • Free VM (Virtual Machine) migration – “Virtualization allows the entire VM state to be transmitted across the network to migrate a VM from one physical machine to another. The communication topology should be designed so as to support rapid virtual machine migration.” (Zhang, 2010)
    • Resiliency – “The network infrastructure must be fault-tolerant against various types of server failures, link outages, or server-rack failures.” (Zhang, 2010)
    • Scalability – “The network infrastructure must be able to scale to a large number of servers and allow for incremental expansion.” (Zhang, 2010)
    • Backward compatibility – “The network infrastructure should be backward compatible with switches and routers running Ethernet and IP.” (Zhang, 2010)
  • Modular data centre (MDC) is another area of fast innovation within the industry, using thousands of servers to form the network infrastructure, being interlinked by switches. These aid in rapid response times as well as helping reduce redundancy. (Zhang, 2010)
  • Distributed file system over clouds such as Google File System (GFS) divide files into chunks of 64 megabytes to provide effective and dependable access to data, adjusted for running on data centres giving extremely high data throughputs with low latency and failures. (Zhang, 2010)
  • Distributed application framework over clouds – “HTTP-based applications usually conform to some web application framework such as Java EE. In modern data centre environments, clusters of servers are also used for computation and data-intensive jobs such as financial trend analysis, or film animation.” (Zhang, 2010)
Popular and dominant cloud computing products are:
  • Amazon Web Services (AWS)
  • Microsoft Windows Azure platform
  • Google App Engine
As this article is from 2010, some of the details may be out of date, but can assume that the points made still stand as it is referring to the key technologies that make up cloud computing, there just may be some more recent ones missing.

What can be done now?

There are many things that cloud computing can do at the moment. Some of these (but by no means all of them) are:

  • Cloud Storage – Services like Dropbox, Google Drive and OneDrive allow for the storage, access, and retrieval of files via an Internet enabled user interface. (IBM Cloud Team, IBM Cloud, 2020)
  • Data Backup – The cloud offers a convenient alternative to using physical mediums such as hard drives to back up data for retrieval should the original data be lost. (IBM Cloud Team, IBM Cloud, 2020)
  • Big Data Analytics - By utilising cloud computing there is the ability to use big data analytics to leverage massive amounts of both structured and unstructured data to harness the benefit of obtaining business value such as evaluating customer buying habits to create targeted advertising and marketing. (IBM Cloud Team, IBM Cloud, 2020)
  • Infrastructure as a service (IaaS) - IaaS comprises of the main building blocks for cloud IT. Generally, it offers access to networking features, computers (virtual or on dedicated hardware), operating systems and data storage space on a pay-as-you-go-basis. (Amazon Web Services, 2022) (Microsoft Azure, 2022)
  • • Platform as a service (PaaS) - PaaS refers to cloud computing services that provide an on-demand setting for developing, testing, delivering, and managing software applications. PaaS is designed to eliminate the need for the organisation of underlying infrastructure (usually hardware and operating systems), and lets developers focus on the deployment and running of web or mobile apps, without worrying about setting up the servers, storage, network, and databases needed. (Amazon Web Services, 2022) (Microsoft Azure, 2022)
  • Software as a service (SaaS) - SaaS is a method for providing software applications over the Internet, on request and typically on a subscription basis. With SaaS, cloud providers host and manage the software application and core infrastructure, and handle any upkeep, like software upgrades and security patching. Users can log into and use a SaaS application from any compatible device over the Internet. The actual application runs in cloud servers that may be far removed from a user's location. A SaaS application may be accessed through a browser or through an app. Online email applications such as Gmail and Office 365, are common examples of SaaS applications. (Amazon Web Services, 2022) (Cloudflare, 2022) (Microsoft Azure, 2022)
Other as-a-service cloud offerings include (but aren’t limited to):
  • Function as a Service (FaaS)/Serverless Computing
  • Artificial Intelligence as a Service (AIaaS)
  • Network as a Service (NaaS)
  • Workspace as a Service (WaaS)
  • Database as a Service (DBaaS)
  • Firewall as a Service (FWaaS)
  • Identity as a Service (IDaaS)

What is likely to be able to do be done soon (say in the next 3 years)?

Some of the things that cloud computing could likely be able to do or facilitate in the near future are:

  • Quantum computing – Quantum computers use the tenets of quantum physics to execute intricate algorithm calculations and handle huge datasets in rapid time. In quantum computing, data is stored in qubits, making the process much faster, as qubits are a simpler form of data.
  • Automation – The cloud can be used to handle tedious and time-consuming tasks which both frees up staff from performing the tasks manually and lowers the cost of operations by requiring less staff overall. (Saleem, 2020)
  • Internet of Things (IoT) – IoT devices can utilise cloud computing as it offers high speed, performance, flexibility, and sufficient storage space to keep the data safe, find resources, and share information among different users within the same space. (Saleem, 2020)
  • Modular Software Development – To be able to utilize the cloud technology, code and applications have to break down into smaller components which are not connected with one another. This makes it simpler for developers to upload their works into the cloud and makes accessing it from remote locations easier too.
  • Improved performance – As the use of cloud computing and the amount of ways in which it is used increase, the speed of the networks and how efficiently data is analysed, received and delivered will also increase to meet user expectations and demands. (DataFlair , 2022)
