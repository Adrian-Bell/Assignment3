Summary

As in Assignment 2, we have created a GitHub repository for the storage and development of the website component of the assignment. This was forked from the repository we created for Assignment 2 and the necessary changes were made to the files contained therein. This included deleting the majority of the report content, creating the relevant files for this assignment, and updating the general look of the website to give it a fresh appearance for the new report. All group members were invited as collaborators, and individuals will again be responsible for committing their content to the repository.

Our audit trail on GitHub is more controlled and constant than Assignment 2. Crunch is still visible close to the due date of the assignment, but a lot of the commits come in around the final week as tasks are finalised. There are large commits throughout the entire assignment. Each person has uploaded their own content, and the summaries of each commit have been more descriptive of the changes taking place in this assignment. I attribute this more even distribution to the implementation of a master word document through which we all worked and drafted our ideas, speeding the deployment of the website content.
